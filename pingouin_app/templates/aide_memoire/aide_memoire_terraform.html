Aide Mémoire terraform

Prérequis :
On considere que le provider CLI est déjà configuré avec un ID et le reste de la sécurité.
Amazon CLI
google cloud

On crée un fichier variables.tf avec les ressources nécéssaires :

variable "project"{
  type = string
  default = "nom_projet"
}

variable "region"{
  type = string
  default = "region_deploie"
}

variable "zone"{
  type = string
  default = "zone_deploi"
}

On pourras acceder à ces variable avec l'objet var. par exemple var.region.

On crée un fichier main.tf avec les ressources nécéssaires :
par exemple :
terraform {
  required_providers {
    aws = {
     source = "hashicorp/aws"
     version = "~> 4.16"
   }
 }

  required_version = ">= 1.2.0"
  }

  provider "aws" {
    region = var.region"
 }

 resource "aws_instance" "app_server" {
   ami = "ami-830c94e3"
   instance_type = "t2.micro"

   tags = {
   Name = "ExampleAppServerInstance"
 }
output var.region }
On décrie ici le provider aws avec l'AMI que l'on veux, la region et le type d'instance.

Si besoin la commmande output peux vous servir à faire sortir une variable (comme print ou echo).

On initialise terraform avec la commande :
terraform init

On peux ensuite faire un controle du code avec la commande validate:
terraform validate

On peux voir une preversion de l ínfra avec la commande plan :
terraform plan

On peux ensuite appliqué (avec cet exemple sur aws) :
terraform apply

On peux detruire ces instances avec la commande :
terraform destroy

Pour mieux diviser son code vous pouvez par exemple mettre les providers dans un fichier providers.tf .
On peux créer aussi un fichier prod.tfvars pour y mettre les variables. Un peu comme un .env en python.
exemple :
ma_variable = "machainedecaractere"
Si vous y mettez des données sensibles, evitez de partager le code. Ajouter le fichier au .gitignore .
terraform output vous permets de recupérer vos outputs.
l'option -raw permets d'enlevé les guillemets.
l'option -json permets de faire un format json
terraform output -raw
terraform output -json
