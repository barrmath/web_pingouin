Aide Mémoire Ansible

Ansible : vous permets déxecuter sur un ou plusieurs ordinateur distant des commandes.
par exemple des apt-get pour installer divers logiciels.

On a pas besoin d'installer Ansible sur les ordinateurs distants. Il faut juste un ordinateur avec ansible d'installé.
Premiére étape : inventoriez les ordinateurs ou servers à automatiser.
crée un fichier .ini par exemple inventaire.ini avec les nom des configs par exemple :

fedoraBDD
fedoraweb

Il faut ensuite modifier le fichier /etc/host pour ajouter les lien entre les nom et les adresses IP.
Par exemple :
#pour faire des test avec ansible et des VMs
0192.168.1.141 fedoraBDD
192.168.1.140 fedoraweb
Vous pouvez verifier si vos serveurs repondent à ansible avec la commande ping :
ansible -i inventaire.ini -m ping --user tux --ask-pass all
Si la commande marche pas : il faut peut être sur le serveur ansible ssh et ssh_pass pour permettre à ansible de fonctionner

Bon le ssh avec un mots de passe il parait que c'est pas assez sécurisé. Donc on va passé par un jeux de clé publique. Génération de clé avec la commande ssh :
ssh-keygen -t ecdsa
Puis mise en place des clés pour les hosts:
ansible -i inventaire.ini -m authorized_key -a 'user=tux state=present key="{{lookup("file", "/home/tux/.ssh/id_ecdsa.pub") }}"' --user tux --ask-pass --become --ask-become-pass all
Maintenant, il faut configurer les roles et les appliqué aux bon serveurs.

Un rôle est une serie d'intruction pour arriver à faire une choses : par exemple installer apache.
Ce role peut être repris plusieur fois si on dois par exemple installer apache sur plusieur machine
On va mettre les rôles dans un repertoire roles.(dossier par defaut de ansbile)
Une fois dans le dossier roles on peut initier une arborescence avec la commande :
ansible-galaxy init nom_du_role
exemple de repertoire :

roles
├── apache
│   ├── handlers
│   │   └── main.yml
│   └── tasks
│   ├── main.yml
│   └── php-install.yml
├── mariadb
│   └── tasks
│   └── main.yml

ici on a 2 rôles apache et mariadb. Mairiadb est le rôle le plus simple : il n y a que des taches (tasks).
Apache est plus complexe avec une tache principale main.yml et une autre as inclure php-install.yml.
Dans le dossier apache on peut aussi voir un dossier handler qui gere le redemarrage de apache en cas de changement.

exemple de fichier role:
---

#1. Cette tâche permet d’installer Apache (httpd) à l’aide du module apt
- name: "apache installation"
  yum:
    name: "httpd"
    state: "present"

#2. Cette tâche active le service Apache
- name: "apache service activation"
  service:
  name: "httpd"
  state: "started"
  enabled: yes

#3. Cette tâche fait appel à un autre fichier de configuration pour installer PHP. Elle est exécutée uniquement si la variable php_install est à vraie (par défaut, elle est à faux)
- name: "install php packages"
  include_tasks: "php-install.yml"
  when: php_install|default(False)|bool

L'indexation est important comme en python.
L'indexation se fait avec 2 expaces.
Le fichier débute par ---.
